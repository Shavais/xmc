6. SUBSCRIBER INDEX (IMPLEMENTED)
===================================

SubscribeOrResolve is implemented in SymbolTable.h/cpp. When an Ident node's Morpher task
cannot find the referenced symbol (because the declaring file's Morpher hasn't run yet),
it registers a continuation instead of reporting an error.

SymbolEntry (SymbolTable.cpp, internal) has:
    std::vector<std::function<void(Symbol*)>> waiters;

SymbolTable::SubscribeOrResolve (SymbolTable.cpp):
    Takes the shard write lock; does the same prefix-match resolution as ResolveSymbol.
    If resolved: returns the symbol (caller sets declaredSymbol and propagates normally).
    If not: interns the name (creating the entry if new via InternStringUnderLock),
            pushes onResolved into entry.waiters, returns nullptr.

SymbolTable::InternSymbol:
    After pushing the new Symbol into entry.candidates, if outWaiters is non-null,
    moves entry.waiters into *outWaiters before releasing the lock.
    Caller (DeclareSymbol in Morpher.cpp) submits each waiter as a pool.detach_task.

Morpher.cpp ExprRule::Ident:
    Symbol* sym = symbols.SubscribeOrResolve(name, path.data(), path.size(),
        [n, &xmo, &symbols, &job, &pool](Symbol* s) {
            n->declaredSymbol = s;
            if (s->declNode) CopyTypeShape(n, s->declNode);
            else n->baseType = s->baseType.load(std::memory_order_relaxed);
            PropagateUp(n, xmo, symbols, job, pool);
        });
    if (!sym) return false;  // deferred; ApplyRule returns false; MorphNoun skips PropagateUp
    // ... set declaredSymbol, copy type, fall through to PropagateUp normally

ApplyRule (and ApplyExprRule, ApplyNodeRule) return bool:
    true  = rule applied, caller should PropagateUp
    false = deferred (Ident subscribed); callback will call PropagateUp when symbol arrives


7. CONCURRENCY MODEL
=====================

One pool task per leaf node for Morpher; one pool task per file for Reviewer/Coder/Emitter.
Morpher tasks are IMPLEMENTED. Reviewer/Coder/Emitter Phase 1 tasks are the next steps.

7a. Concurrent Morphing per noun (IMPLEMENTED)
------------------------------------------------
Leaf MorphNoun tasks are submitted by the Parser during Drive(), not by MorphTree afterward.

  Parser.cpp addLeaf():
    pool.detach_task([leaf, &xmo, &symbols, &job, &pool]() {
        Morpher::MorphNoun(leaf, xmo, symbols, job, pool);
    });

  Parser.cpp popNode() for interior nodes (childCount > 0):
    n->parser_complete.store(true, std::memory_order_seq_cst);
    if (n->morphed_children.load(std::memory_order_seq_cst) == n->childCount)
        if (!n->morphed.exchange(true, std::memory_order_acq_rel))
            Morpher::MorphNoun(n, xmo, symbols, job, pool);  // inline, no new task

  Three-field protocol on ParseTreeNode:
    morphed_children  atomic<uint32_t>  incremented seq_cst by each child's PropagateUp
    parser_complete   atomic<bool>      stored true seq_cst by popNode() when node closes
    morphed           atomic<bool>      test-and-set; winner runs this node's morph rule

  The seq_cst ordering on the store/load pair eliminates the lost-wakeup race: if the
  parser closes a node before the last child's morpher increments morphed_children, the
  parser sees the complete count and wins morphed; if the morpher increments first, it
  sees parser_complete=true and wins morphed. Exactly one thread processes each node.

  MorphNoun(node, xmo, symbols, job, pool):
    if (ApplyRule(node, ...)) PropagateUp(node, ...);
    // ApplyRule returns false for deferred Idents; callback calls PropagateUp directly.

  PropagateUp walks parent pointers, applying the same morphed_children / parser_complete /
  morphed.exchange protocol at each ancestor inline (no new pool tasks for interior nodes).

  Compiler.cpp RunModifiedPipeline:
    fileFutures.wait();   // parser tasks complete (morpher tasks may still run)
    pool.wait();          // drain all remaining morpher and subscriber callback tasks
    // then Reviewer, Coder, Emitter Phase 1 run (sequentially for now)

  MorphTree (Morpher.cpp): only writes the morpher log if requested. All morphing is done
  by MorphNoun. MorphTree takes no leaves parameter.

7b. Concurrent Reviewing per file
----------------------------------
In Compiler.cpp RunReviewer, replace the sequential loop with parallel pool tasks:

    BS::multi_future<void> futures;
    for (auto& xmo : s.xmos) {
        Xmo* x = xmo.get();
        futures.push_back(pool.submit_task([&s, x] {
            if (!s.job.ErrorOccurred.load(std::memory_order_relaxed))
                Reviewer::Review(*x, s.symbols, s.job);
        }));
    }
    futures.wait();

  Safety: Reviewer only reads from the parse tree (written by Parser/Morpher, now complete).
  It writes only its own xmo's nodes (paramSlot, totalStackSize, funcFlags) and only the
  symbols it owns (Symbol::offset). No two Reviewer tasks share an xmo or its symbols.
  The SymbolTable is read-only after Morpher convergence.

8. SETTLED DESIGN DECISIONS (do not relitigate)
=================================================

Parser submits leaf MorphNoun tasks during Drive():
  addLeaf() calls pool.detach_task immediately. popNode() closes each interior node and
  applies the morphed_children / parser_complete / morphed.exchange protocol to decide
  whether to also run MorphNoun inline. This replaced the old "collect leaves, then drain"
  design. DO NOT revert to calling MorphTree to submit tasks after Drive() returns.

Three-field morphing protocol instead of pendingChildren:
  ParseTreeNode has morphed_children (atomic<uint32_t>), parser_complete (atomic<bool>),
  morphed (atomic<bool>). pendingChildren is gone. The seq_cst store/load pair is required
  to prevent lost wakeups — do not downgrade to relaxed or acq/rel ordering on those two ops.

pool.detach_task for fire-and-forget morpher tasks:
  pool.submit_task is [[nodiscard]] — the returned future must be stored or the compiler
  warns. Use pool.detach_task for leaf morpher tasks and subscriber callbacks.

No pool.wait() from inside a pool task:
  Parser runs as a pool task. pool.wait() from within it deadlocks or returns early.
  pool.wait() is called from the main thread (RunModifiedPipeline) after fileFutures.wait().

Symbol::declNode on Symbol:
  When DeclareSymbol creates a symbol, it stores sym->declNode = node.
  Ident and Call nodes read sym->declNode to copy type shape. Required for concurrent Morpher.

No morphQueue on Xmo:
  Removed. No leaves vector either — leaf tasks are submitted inline from addLeaf/popNode.

String literal .rdata symbols use IMAGE_SYM_CLASS_STATIC + per-xmo localStrMap:
  String literals are function-local; their COFF backing symbols must not be external.
  EmitPhase2 creates a fresh localStrMap per xmo. Phase 1 stores the bare variable name
  as the relocation target; Phase 2 looks it up in the per-xmo map. This avoids both
  cross-xmo name collisions and LNK2005 multiply-defined errors.

Emitter Phase 2 is single-pass:
  All xmos' funcSymbols are defined (or upgraded) before their code bytes are written,
  because the loop processes each xmo's strings, then symbols, then code, then relocs in
  one pass. No separate pre-pass needed.

CRT = none for test projects:
  hello.xmc and multifile.xmc both use CRT=none. The Linker emits /ENTRY:main /NODEFAULTLIB.

Windows x64 only:
  All register names (rcx/rdx/r8/r9), ABI rules (shadow space, 16-byte alignment),
  and COFF output are Windows x64 specific.

========================================

1. Working in a git worktree instead of the main project directory.
   Always work at C:\Shavais\Projects\C++\xmc\ directly.

2. Running "xmc.exe hello.xm" or "xmc hello.xmc" — the argument is the base name only.
   Run from inside the test directory: xmc.exe hello

3. Reverting to the old "collect leaves in Drive(), drain in MorphTree" design.
   Leaf tasks are submitted from addLeaf()/popNode() during Drive(). MorphTree only
   writes the log. Do not add leaf submission back to MorphTree.

4. Using pool.submit_task for fire-and-forget morpher or callback tasks.
   submit_task is [[nodiscard]]; use pool.detach_task instead.

5. Calling pool.wait() from within a pool task — deadlock risk.
   pool.wait() belongs in RunModifiedPipeline on the main thread, after fileFutures.wait().

6. Adding a declSite map to Morpher — use sym->declNode instead (already on Symbol).

7. Re-adding morphQueue or a leaves vector to Xmo or ParseCtx — both were deliberately
   removed. Leaf tasks are submitted inline; no collection step exists.

8. Adding external symbol entries for project-local function calls in Emitter Phase 2.
   Project-local functions (those in any xmo's funcSymbols) use defined symbol indices.

9. Implementing the subscriber callback as a blocking wait — it must be a non-blocking
   continuation submitted as a pool.detach_task when the declaration fires.

10. Using IMAGE_SYM_CLASS_EXTERNAL for string literal .rdata symbols — causes LNK2005
    when two xmos have same-named locals (e.g. both have "msg"). Use IMAGE_SYM_CLASS_STATIC
    and a per-xmo localStrMap in EmitPhase2.

11. Looking up string literal COFF symbols in the cross-xmo symbolMap — they live in the
    per-xmo localStrMap. Only function call relocations (kind=0) use symbolMap.
